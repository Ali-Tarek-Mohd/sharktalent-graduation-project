# SharkTalent Freelance Marketplace

SharkTalent is a graduation project developed as a freelance marketplace platform concept. The system is designed to connect clients with freelancers in a more structured, transparent, and organized way.

The main idea behind SharkTalent is to reduce common problems found in freelance platforms, such as spam proposals, unclear freelancer selection, weak project tracking, and lack of transparency between clients and freelancers.

## Project Overview

SharkTalent allows clients to post projects and receive proposals from freelancers. Freelancers can create profiles, apply to suitable projects, and be evaluated based on their skills, ratings, and project history.

The platform includes a rule-based recommendation concept that helps clients understand why certain freelancers may be suitable for a project. This makes the selection process more transparent compared to platforms where recommendations are not clearly explained.

The system also includes proposal limits to reduce spam and encourage freelancers to apply only to projects that match their skills.

---

## Problem Statement

Many freelance platforms provide useful services, but they can also create difficulties for both clients and freelancers.

Some common issues include:

- Clients may receive too many low-quality or unrelated proposals.
- Freelancers may apply to many projects without properly matching the client’s needs.
- New freelancers may struggle to stand out.
- Clients may not clearly understand why a freelancer is recommended.
- Communication and project tracking can become disorganized.
- Ratings may not always reflect real project performance.

SharkTalent was designed to address these issues by creating a more controlled and transparent freelance marketplace structure.

## Key System Concepts

### Proposal Limit System

SharkTalent includes a proposal limit to reduce spam applications. Instead of allowing freelancers to apply to unlimited projects, the system limits the number of active proposals a freelancer can submit.

This encourages freelancers to apply more carefully and helps clients receive more relevant proposals.

### Rule-Based Freelancer Recommendation

The platform uses a rule-based recommendation concept instead of an artificial intelligence system. The recommendation is based on clear factors such as:

- Freelancer skills
- Project category
- Previous ratings
- Profile completeness
- Relevant experience
- Proposal quality

This makes the system easier to understand and more transparent for users.

### Transparent Selection Process

Clients should be able to understand why a freelancer is suitable for a project. SharkTalent focuses on showing clear reasons behind recommendations rather than hiding the selection process.

### Post-Project Rating

Ratings are intended to be given after project completion. This helps make feedback more meaningful and reduces the chance of unrealistic or fake ratings before work is completed.

## Project Workflow

A simple workflow of the system is:

1. User creates an account.
2. User selects account type: client or freelancer.
3. Client posts a project with details and requirements.
4. Freelancer browses available projects.
5. Freelancer submits a proposal.
6. System checks proposal limit.
7. Client reviews proposals.
8. Client selects a freelancer.
9. Project begins and progress is tracked.
10. Project is completed.
11. Client gives freelancer a rating.
